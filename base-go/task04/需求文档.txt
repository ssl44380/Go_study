设计一个个人博客系统的后端

数据库：
    User 用户表 字段有 id name password age  email  创建时间 更改个人信息时间  删除时间  关联Post文章表  文章数量 实现文章新增或者删除时文章数量对应增减 注销用户时，删除用户的所有文章 删除用户可以用软删除
    post 文章表  字段有 id title content 创建时间 更新时间 删除时间 评论数量 评论列表 关联comments表 实现评论熟练新增或者删除时评论数量对应增减 删除文章时，删除文章的所有评论  删除文章可以用软删除
    Comment 评论表 字段有  id content 创建时间 删除时间 更新时间（更新接口一般不对外开放） 该模型可以引用自身实现评论的评论  大评论删除的时候，内部的小评论也需要全部删除  内部小评论删除时大评论不用删除

用户：
    author 作者，管理员用户，需要认证，可以删除文章，新增文章，修改文章，注销自己的账号
    tourist 游客，非管理员用户，只能看文章并查看所有人的评论

认证和授权 
    用户注册时登录，需要对用户密码进行加密存储，登录验证时需要验证用户输入的用户名和密码
        如果用户名不存在则提示，该用户未注册
        如果密码出错，则提示，密码输入错误
        使用JWT进行用户认证和鉴权
        实现用户注册和登录功能，用户注册时需要对密码进行加密存储，登录时验证用户输入的用户名和密码。
        使用 JWT（JSON Web Token）实现用户认证和授权，用户登录成功后返回一个 JWT，后续的需要认证的接口需要验证该 JWT 的有效性。
文章管理功能
    实现文章的创建功能，只有已认证的用户才能创建文章，创建文章时需要提供文章的标题和内容。
    实现文章的读取功能，支持获取所有文章列表和单个文章的详细信息。
    实现文章的更新功能，只有文章的作者才能更新自己的文章。
    实现文章的删除功能，只有文章的作者才能删除自己的文章。
评论功能
    实现评论的创建功能，已认证的用户可以对文章发表评论。
    实现评论的读取功能，支持获取某篇文章的所有评论列表。

日志功能
    报错时，抛出相应的提示错误
